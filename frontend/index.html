Index Architecture

  • Composite Sort Key: The composite sorting index (created_at DESC, id DESC) allows the query planner to do a direct index-lookup and read exactly LIMIT rows without sorting in memory.
  • Filtered Seek: Category composite index (category, created_at DESC, id DESC) enables multi-column B-Tree seeks, completely bypassing product rows belonging to other categories.
